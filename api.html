Server launcher

niimblue.start_server(port=5000, host='127.0.0.1', extra_args=None, wait=1.0, **popen_kwargs)[source]

Launch a niimblue-cli server subprocess.

The function starts the server in the background and returns the subprocess.Popen object so the caller can manage its lifetime.

Parameters:

  • port (int) – TCP port for the REST server to listen on (default 5000).

  • host (str) – Host/interface to bind to (default "127.0.0.1").

  • extra_args (Optional[list[str]]) – Additional command-line arguments forwarded verbatim to niimblue-cli server.

  • wait (float) – Seconds to sleep after spawning the process to give the server time to become ready (default 1.0). Pass 0 to skip.

  • **popen_kwargs (Any) – Keyword arguments forwarded to subprocess.Popen.

Return type:

Popen

Returns:

The running server process.

Raises:

FileNotFoundError – If niimblue-cli is not found on PATH.

Example:

proc = start_server(port=5000)
try:
    with Printer("http://localhost:5000") as p:
        p.connect(Transport.SERIAL, "COM8")
        p.print(PrintJob(image_source="label.png"))
finally:
    proc.terminate()
    proc.wait()

class niimblue.Printer(base_url='http://localhost:5000', timeout=10.0)[source]

Bases: object

High-level client for the niimblue-cli REST server.

Wraps every REST endpoint in a typed Python method. Can be used as a context manager — __exit__() disconnects automatically.

Parameters:

  • base_url (str) – URL of the running niimblue-cli server (default: "http://localhost:5000").

  • timeout (float) – Seconds to wait for each HTTP response (default: 10).

Example:

with Printer("http://localhost:5000") as p:
    p.connect(Transport.BLE, "AA:BB:CC:DD:EE:FF")
    info = p.info()
    print(info.model, info.charge)
    p.print(PrintJob(image_source=Path("label.png")))

Print job

class niimblue.PrintJob(image_source=None, print_direction=PrintDirection.LEFT, label_width=None, label_height=None, image_position=ImagePosition.CENTRE, image_fit=ImageFit.CONTAIN, quantity=1, density=3, threshold=128, print_task=None, label_type=1)[source]

Bases: object

Parameters for a single print job.

The image_source field is flexible and accepts several types:

  • str — local file path or remote URL. If the path exists on disk it is read as a file; otherwise the string is sent to the server as a URL.

  • pathlib.Path — local image file (PNG, JPEG, BMP, …).

  • bytes — raw image bytes.

  • PIL.Image.Image — an already-opened Pillow image object (requires the pillow extra).

Parameters:

  • image_source (Union[Path, str, bytes, Any, None]) – Image to print. See above for accepted types.

  • print_direction (PrintDirection) – Feed direction / orientation of the label.

  • label_width (Optional[int]) – Label width in pixels. Auto-detected when None.

  • label_height (Optional[int]) – Label height in pixels. Auto-detected when None.

  • image_position (ImagePosition) – Anchor point of the image within the label.

  • image_fit (ImageFit) – Resizing strategy.

  • quantity (int) – Number of copies to print.

  • density (int) – Print density (darkness). Typical range 1–5.

  • threshold (int) – Binarisation threshold for dithering (0–255, default 128).

  • print_task (Optional[str]) – Hardware print-task code (e.g. "B1"). Auto-detected by the server when None.

  • label_type (int) – Label type identifier sent to the firmware.

Bases: str, Enum

Strategy used to fit the image into the label dimensions.

CONTAIN

Scale to fit, preserving aspect ratio (default).

COVER

Scale to fill, cropping if necessary.

FILL

Stretch to exactly fill (ignores aspect ratio).

INSIDE

Like CONTAIN but never upscales.

OUTSIDE

Like COVER but never downscales.
